# 📂 Корпоративная система управления кадровой документацией

Программный комплекс на базе **C# Windows Forms**, разработанный в рамках производственной практики ПП.02.01. Приложение представляет собой многодокументную графическую среду (MDI) для автоматизации процессов учета, редактирования и выгрузки аналитической отчетности кадровой службы предприятия.

Архитектура проекта спроектирована в строгом соответствии с методологией **MVC (Model-View-Controller)** и принципами **SOLID**.

---

## 🛠 Технологический стек

* **Язык программирования:** C# (.NET Framework 4.8 / .NET 8)
* **Графический интерфейс:** Windows Forms (WinForms)
* **Управление пакетами:** NuGet Package Manager
* **Сторонние библиотеки:** 
  * `Newtonsoft.Json` (высокопроизводительная сериализация данных)
  * `iTextSharp` (генерация PDF-документов)

---

## 🏗 Архитектурные паттерны и паттерны проектирования

В ходе реализации индивидуального задания были интегрированы следующие паттерны:

* **MVC (Model-View-Controller):** Полное разделение проекта на изолированные слои. Управление логикой полностью вынесено из кода форм в контроллер `DocumentController`.
* **Memento (Снимок):** Реализован класс `UndoManager`, который сохраняет снимки текстовых состояний во внутренний стек. Обеспечивает функционал отмены действий (`Undo` / `Ctrl+Z`).
* **Observer (Наблюдатель):** Событийная модель взаимодействия. Контроллер генерирует события (`DocumentCreated`, `StatusChanged`), на которые подписывается главная форма для динамического обновления интерфейса.
* **Factory (Фабрика):** Класс `ReportFactory` инкапсулирует алгоритмы создания табличных отчетов в формате PDF, скрывая внутреннюю сложность работы с библиотекой `iTextSharp`.
* **Strategy (Стратегия):** Используется для гибкого переключения алгоритмов фильтрации и поиска кадровых документов через `BindingSource`.

---

## 🌟 Ключевой функционал

* **Ролевая модель доступа (RBAC):** Динамическое перестроение интерфейса в зависимости от роли пользователя (Администратор или Пользователь).
* **Многодокументный интерфейс (MDI):** Возможность одновременной работы с каскадом текстовых документов внутри единого родительского окна.
* **Персистентное хранилище (JSON):** Автоматическое сохранение базы документов в локальный файл при закрытии приложения и по горячим клавишам.
* **Контроль несохраненных изменений:** Валидация состояния формы перед закрытием (флаг `IsDirty`) для предотвращения случайной потери данных.
